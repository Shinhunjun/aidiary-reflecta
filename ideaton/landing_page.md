# 🎯 랜딩페이지 제작 가이드 (`landingpage_instructions.md`)

이 문서는 **아이디어톤 제출용 랜딩페이지 제작에만 초점을 맞춘 가이드**입니다.  
이미 서비스를 개발하거나 운영 중인 팀도 그대로 활용할 수 있도록 설계되었습니다.

---

## 📌 1. 랜딩페이지 목적

랜딩페이지는 **심사위원과 사용자에게 서비스를 한눈에 이해시키는 단일 웹페이지**입니다.

- **핵심 질문에 답해야 함:**
  1. 이 서비스는 **누구의 어떤 문제**를 해결하는가?  
  2. 그 문제를 **어떻게 해결**하는가?  
  3. 왜 **우리 팀의 접근 방식이 차별화**되는가?  
  4. 사용자는 **무엇을 해야 하는가? (CTA)**

> ✅ **목표:** 30초 안에 “무슨 서비스인지” 직관적으로 이해시키기

---

## 🧱 2. 페이지 기본 구조

| 섹션 | 주요 내용 | 비고 |
|------|-----------|------|
| **Hero Section (첫 화면)** | 서비스명 / 한 줄 슬로건 / 대표 이미지 / CTA 버튼 | 첫인상 결정 |
| **Problem → Solution (문제와 해결)** | 문제 정의, 해결 방법, 서비스의 가치 제시 | ‘왜 필요한가’를 설명 |
| **Core Features (핵심 기능)** | 3~4개 기능 요약 (짧은 문장 + 아이콘/이미지) | 직관적으로 전달 |
| **User Flow (사용 흐름)** | 와이어프레임 or 스토리보드 이미지 삽입 | 필수 포함 |
| **Tech Stack / Demo** | 기술 스택, 실제 배포 링크 or 데모 영상 | 이미 배포된 경우 |
| **Team & Contact** | 팀 구성 / 역할 / 문의 이메일 | 신뢰 요소 |
| **CTA (Call To Action)** | “서비스 체험하기”, “대기명단 등록” 등 하나의 버튼 | 페이지 하단에 반복 |

---

## 🧩 3. 운영 중인 서비스의 경우

이미 실제 서비스가 있다면 다음 중 한 가지 방법으로 진행하세요.

| 방식 | 설명 | 예시 |
|------|------|------|
| **① 별도 랜딩페이지 제작 (권장)** | 서비스 메인과 분리된 홍보용 페이지 | `https://서비스도메인/landing` |
| **② 노코드 툴 활용** | 빠른 제작용 | Notion, Slashpage, Typed, Carrd, Wix 등 |
| **③ 메인 페이지 일부 수정** | 기존 서비스에 ‘소개 섹션’ 추가 | Hero + About + CTA만 추가 |

## 🧭 4. 콘텐츠 작성 팁

### 🔹 Hero Section
- **한 줄 슬로건**: “한 문장으로 설명 가능한가?”
- **CTA 버튼**: “바로 체험하기”, “서비스 둘러보기” 등 하나만.
- **배경 이미지**: 실제 서비스 화면 캡처 or 핵심 가치 시각화

### 🔹 Problem & Solution
```text
❌ 많은 대학생이 새 친구를 사귀기 어렵습니다.
✅ UFO는 익명 기반의 친구 매칭 서비스를 통해 대학생 간 연결을 돕습니다.
````

### 🔹 Features (예시)

* 👤 **프로필 탐색** – 주변 대학생을 빠르게 탐색
* 💬 **1:1 채팅** – 안전한 익명 대화 환경
* 🎯 **추천 알고리즘** – 관심사 기반 친구 추천
* 🕒 **이벤트 매칭** – 오프라인 만남 확장 기능

### 🔹 Prototype / Storyboard

* 와이어프레임 이미지 2~3장 첨부
  (모바일/웹 화면, 사용자 흐름)

### 🔹 Tech Stack

> React / Express / MySQL / AWS / Vercel
> v1.2.0 배포 완료, 향후 채팅 추천 알고리즘 고도화 예정

### 🔹 CTA

* “👉 [서비스 둘러보기](https://yourservice.link)”
* “📱 [앱 다운로드](https://yourservice.link/app)”
